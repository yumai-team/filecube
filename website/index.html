FileCube 如何工作？

三个核心概念，彻底改变你的文件管理方式

1

导入即登记

把文件"导入"FileCube，文件内容会被计算 SHA-256 哈希，并以 UUID 重命名后存入专用目录。原始路径和标签信息记录在本地的 SQLite 数据库中。

2

标签代替目录

给你的文件打上任意多个标签（如"工作"、"设计稿"、"2024"）。筛选时只需勾选标签，文件立刻呈现，无需记忆目录结构。

3

导出即还原

需要使用时，从 FileCube"导出"文件，自动恢复原始文件名。别人直接打开存储目录，看到的只是一堆 UUID 命名的文件，隐私天然保护。

FileCube 从架构层面保护你的数据安全

🔐 纯本地运行

FileCube 无任何网络通信功能。所有数据存储在本地 SQLite 数据库，不联网、不上传、不追踪。你拥有数据的完全控制权。

原始文件名仅存储于本地数据库，与存储文件完全分离。即便存储设备丢失，他人也无法通过文件名推测文件内容。

常见问题

FileCube 是免费的吗？
是的，FileCube 完全免费，无广告、无内购、无追踪。未来计划发布的 v1.0 正式版也将保持免费。
我的文件会被上传到云端吗？
不会。FileCube 无任何网络通信功能，所有数据存储在本地。你的文件只存在于你的电脑上，我们看不到、也拿不到。
绿色版和安装版有什么区别？
目前只提供绿色版。绿色版解压即用，无需安装，可以放在 U 盘里随身携带。所有数据和设置存储在程序所在目录，换电脑时直接复制整个文件夹即可迁移。
我的文件导入后，原始文件还在吗？
导入时 FileCube 会将文件复制到专用存储目录（以 UUID 重命名）。默认不会删除原始文件，你可以手动选择是否删除原始文件以节省空间。
如果卸载 FileCube，我的文件会丢失吗？
不会。FileCube 的存储目录中的文件可以通过"导出"功能恢复原始文件名。建议在卸载前先将需要的文件导出。
支持 macOS 或 Linux 吗？
目前仅支持 Windows 10/11。macOS 和 Linux 版本在计划中，取决于用户需求和开发资源。如果你希望看到其他平台的版本，请在 GitHub Issues 中告诉我们！
