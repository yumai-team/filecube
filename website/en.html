How Does FileCube Work?

Three core concepts that change everything about how you organize files

1

Import & Register

"Import" files into FileCube. Each file's SHA-256 hash is computed, then stored under a UUID name in a dedicated directory. Original paths and tags are recorded in a local SQLite database.

2

Tags Instead of Folders

Assign any number of tags to each file ("work", "design", "2024"). Filter by selecting tags — files appear instantly, no folder hierarchy to memorize.

3

Export & Restore

When you need a file, "export" it from FileCube — the original filename is automatically restored. Anyone browsing the storage directory sees only UUID-named gibberish.

Privacy & Security

FileCube has zero network capabilities. All data lives in a local SQLite database — no uploads, no tracking. You're in complete control.

🛡 Storage Obfuscation

Set a password for your FileCube database. Without it, nobody can browse file lists or tags — even if the DB file is copied elsewhere.

Frequently Asked Questions

Is FileCube really free?
Yes, 100% free. No ads, no in-app purchases, no tracking. The upcoming v1.0 release will also remain free.
Will my files be uploaded to the cloud?
Never. FileCube has zero network communication. All data stays on your local machine. We can't see it, and we can't take it.
What's the difference between the ZIP and EXE downloads?
Both are portable (no installer). The ZIP version extracts into a folder — great for USB drives. The EXE is a single standalone file. Functionally identical otherwise.
What happens to my original files after importing?
FileCube copies files into its dedicated storage directory (renamed to UUID). Original files are NOT deleted by default — you can choose to delete originals manually to save disk space.
If I uninstall FileCube, will I lose my files?
No. Files in the storage directory can be restored via the Export function. We recommend exporting important files before removing FileCube.
Does it run on macOS or Linux?
Currently Windows 10/11 only. macOS and Linux versions are planned based on user demand. Let us know on GitHub Issues if you want them!
How do I report bugs or request features?
Visit our GitHub Issues. Every piece of feedback gets read and considered for future releases.
